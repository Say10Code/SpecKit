# ETSI TS 102 223: Card Application Toolkit (CAT)

> **Версия**: V18.2.0 (Release 18), апрель 2025
> **Комитет**: ETSI TC SET (Secure Element Technologies)
> **Страниц**: 252

## Обзор

ETSI TS 102 223 определяет **Card Application Toolkit (CAT)** — механизм, с помощью которого UICC может взаимодействовать с терминалом и сетью **проактивно** (не только отвечать на команды, но и инициировать их). Это стандарт, известный как **SIM Toolkit (STK)** в GSM-контексте и **USAT (USIM Application Toolkit)** в 3GPP-контексте. ^[extracted]

> 💡 **Ключевая идея**: Обычно терминал → команда → UICC. С CAT: UICC говорит терминалу "сделай X" через proactive command, меняя роли.

## Четыре столпа CAT

### 1. Profile Download (Clause 5)
После активации USIM-сессии терминал отправляет команду **TERMINAL PROFILE** — битовую маску, сообщающую UICC, какие CAT-функции терминал поддерживает. 132+ бит (Release 18), покрывающих:
- Display capabilities (размер экрана, атрибуты текста, иконки)
- Proactive commands support (какие команды поддерживаются)
- Event download capabilities (какие события мониторятся)
- Bearer types (CS, GPRS, IMS, eCAT)
- Security (secured commands, secure channel)

### 2. Proactive UICC (Clause 6)
**Проактивные команды** — UICC инициирует действия в терминале. UICC отправляет `'91 XX'` (proactive command pending) в ответ на STATUS → терминал выполняет FETCH для получения команды.

Ключевые proactive commands:

| Команда | Что делает | Пример использования |
|---|---|---|
| **DISPLAY TEXT** | Показать текст на экране | "Добро пожаловать в сеть Vodafone" |
| **GET INKEY** | Запросить 1 символ от пользователя | "Нажмите 1 для меню" |
| **GET INPUT** | Запросить строку ввода | "Введите PIN2" |
| **MORE TIME** | Запросить больше времени на операцию | Длинная крипто-операция |
| **PLAY TONE** | Воспроизвести звук | Beep, dial tone |
| **SET UP MENU** | Создать STK-меню | Главное меню оператора |
| **SELECT ITEM** | Показать список выбора | Выбор услуги из меню |
| **SEND SHORT MESSAGE** | Отправить SMS | Фоновая отправка SMS |
| **SET UP CALL** | Установить звонок | Экстренный вызов |
| **REFRESH** | Сообщить об изменении файлов | После OTA-обновления |
| **SEND DTMF** | Отправить DTMF-тон | Голосовое меню |
| **SET UP EVENT LIST** | Подписаться на события | Мониторинг location update |
| **SEND DATA / RECEIVE DATA** | Передача данных через UICC | Мобильный банк, IoT |
| **LAUNCH BROWSER** | Открыть браузер терминала | Портал оператора |
| **OPEN CHANNEL / CLOSE CHANNEL** | BIP (Bearer Independent Protocol) | Интернет-соединение через UICC |
| **RETRIEVE/SUBMIT MULTIMEDIA MESSAGE** | MMS | Картиночный сервис |
| **COMMAND CONTAINER** | Контейнер для команды другому приложению | eCAT, LSI |
| **ACTIVATE** | Активировать UICC-приложение | Запуск другого приложения |

### 3. Event Download (Clause 7)
UICC подписывается на события через **SET UP EVENT LIST**. Когда событие происходит, терминал уведомляет UICC через **ENVELOPE (EVENT DOWNLOAD)**.

События:
- **MT call** — входящий звонок
- **Call connected / disconnected** — состояние звонка
- **Location status** — изменение location area
- **User activity** — нажатие клавиш пользователем
- **Idle screen available** — экран ожидания свободен (для idle mode text)
- **Browser termination** — браузер закрыт
- **Data available** — данные получены по каналу связи
- **Access Technology Change** — смена RAT (2G↔3G↔4G↔5G)
- **Language selection** — смена языка
- **HCI Connector Event** — contactless событие

### 4. ENVELOPE Commands (Clause 7)
Терминал отправляет ENVELOPE для:
- **Menu Selection** — пользователь выбрал пункт меню
- **Call Control** — UICC модифицирует/блокирует исходящий вызов
- **Timer Expiration** — таймер UICC истёк
- **Event Download** (все типы событий выше)

## BIP (Bearer Independent Protocol, Clauses 6.4.27-6.4.31)

BIP позволяет UICC открывать TCP/UDP соединения через терминал:
- **OPEN CHANNEL**: CS (CSD), GPRS, Local Bearer, Default Bearer
- **UICC Server Mode** / **Terminal Server Mode**
- **CLOSE CHANNEL**, **SEND DATA**, **RECEIVE DATA**, **GET CHANNEL STATUS**

Используется для: веб-сервера на UICC (SCWS), банковских апплетов, IoT.

## Структура Proactive Command (BER-TLV)

```
Tag: D0 (Proactive UICC Command)
├─ Command Details (Tag '81'): номер команды, тип, квалификатор
├─ Device Identities (Tag '82'): источник/назначение (UICC→Terminal)
├─ Command-specific data (разные теги)
│   └─ e.g. DISPLAY TEXT → Text String (Tag '8D'), Alpha Id (Tag '05')
│   └─ e.g. SET UP MENU → Menu items (Tag 'xx'), icons
└─ Возможны дополнительные поля: Icon ID, Text Attribute, Frame ID
```

## Терминальный ответ (Terminal Response)

После выполнения proactive command терминал возвращает результат через **TERMINAL RESPONSE** (FETCH → UICC):

```
Tag: D1 (Terminal Response)
├─ Command Details: номер выполненной команды
├─ Device Identities
├─ Result: '00'=OK, '01'=user abort, '02'=no info, ...
└─ Доп. данные: Text String (GET INKEY result), Channel data, etc.
```

## Вызов Control by NAA

UICC может **модифицировать или блокировать** исходящие вызовы:
- Call Control ENVELOPE отправляется при попытке вызова
- UICC возвращает: allow / modify (другой номер / supplementary service) / disallow
- Используется для: FDN, BDN, corporate policy routing

## CAT в 3GPP контексте

3GPP TS 31.111 (USAT) определяет, какие CAT-функции обязательны для 3GPP-приложений USIM. Основные отличия USAT:
- Некоторые proactive commands могут быть необязательны
- TERMINAL PROFILE битовая маска синхронизирована с EF_UST (USIM Service Table)
- Поддерживается eCAT (extended CAT) для удалённого управления
