# ETSI TS 102 221: UICC-Terminal Interface — Physical and Logical Characteristics

> **Версия**: V18.2.0 (Release 18), июнь 2024
> **Комитет**: ETSI TC SET (Secure Element Technologies)
> **Страниц**: 205

## Обзор

ETSI TS 102 221 — это **базовый стандарт**, определяющий интерфейс между UICC (Universal Integrated Circuit Card) и терминалом. Он не привязан к конкретному поколению сети (2G/3G/4G/5G) — это фундаментальная платформа, на которой работают все приложения, включая USIM, ISIM, SIM.

Стандарт основан на ISO/IEC 7816 серии, но адаптирован для телеком-среды.

## Ключевые разделы

### 1. Физические характеристики (Clause 4)
- **4 form factor'а**: ID-1 (credit-card), Plug-in (2FF), Mini-UICC (3FF), 4FF (nano-SIM)
- UICC может быть съёмной (removable) или встроенной (eUICC, soldered)
- Положение контактов согласно ISO/IEC 7816-2

### 2. Электрический интерфейс (Clause 5)
- **Классы напряжения**:
  - Class A: 5V ± 10% (legacy)
  - Class B: 3V ± 10% (основной)
  - Class C: 1.8V ± 10% (современный)
  - Class D: 1.2V ± 0.1V (Release 17+)
- Контакты: C1=Vcc, C2=RST, C3=CLK, C5=GND, C6=Vpp (опционально), C7=I/O

### 3. Установление связи (Clause 6)
- **Cold Reset**: подача питания → RST low → CLK → RST high → ATR
- **Warm Reset**: сброс без снятия питания
- **ATR (Answer To Reset)**: до 33 байт, содержит параметры протокола, historical bytes, global interface bytes
- **PPS (Protocol and Parameter Selection)**: согласование скорости (Fi/Di) после ATR

### 4. Транспортные протоколы (Clause 7)
- **T=0**: полудуплексный, байт-ориентированный. Использует procedure bytes ('61xx', '6Cxx') + GET RESPONSE
- **T=1**: блочно-ориентированный, полнодуплексный. Блоки: I-block (данные), R-block (квитанции), S-block (управление)
- Поддержка chaining (цепочки блоков), error detection (LRC/CRC)

### 5. Файловая система (Clause 8)
Ключевая архитектурная модель UICC:
- **MF (Master File)** — корень файловой системы
- **DF (Dedicated File)** — директория (может содержать EF и другие DF)
- **ADF (Application DF)** — точка входа в приложение (выбирается по AID)
- **EF (Elementary File)** — содержит данные:
  - Transparent EF — поток байт
  - Linear Fixed EF — записи фиксированной длины
  - Cyclic EF — кольцевой буфер записей
  - BER-TLV EF — TLV-структуры

Выбор файла: по FID (2 байта), по пути (MF→DF→EF), по SFI (Short File Identifier), по AID (для ADF).

### 6. Безопасность (Clause 9)
- **Security attributes**: Access Mode (AM) + Security Condition (SC) для каждого файла
- **Форматы**: Compact (AM + SC байты) и Expanded (BER-TLV DO)
- **Access Rule Referencing (ARR)**: ссылка на запись в EF_ARR через tag '8B'
- **PIN-ы**: Universal PIN, Application PIN, Local PIN
- **Key reference mapping**: PIN1→'01', PIN2→'02', Universal PIN→'11'

### 7. Команды (Clauses 10-12)
- **Generic**: SELECT, STATUS, READ/UPDATE BINARY, READ/UPDATE RECORD, SEARCH RECORD, INCREASE
- **PIN**: VERIFY, CHANGE, DISABLE, ENABLE, UNBLOCK PIN
- **Security**: AUTHENTICATE, GET CHALLENGE, MANAGE SECURE CHANNEL
- **Channel**: MANAGE CHANNEL (логические каналы 0-19)
- **Lifecycle**: ACTIVATE/DEACTIVATE FILE
- **CAT**: TERMINAL PROFILE, ENVELOPE, FETCH, TERMINAL RESPONSE
- **Data**: RETRIEVE DATA, SET DATA
- **Power**: SUSPEND UICC, TERMINAL CAPABILITY

### 8. Application Independent Files (Clause 13)
Файлы на уровне MF, не зависящие от приложений:
- `EF_DIR` (0x2F00) — список AID приложений
- `EF_ICCID` (0x2FE2) — уникальный идентификатор карты
- `EF_PL` (0x2F05) — предпочитаемые языки
- `EF_ARR` (0x2F06) — Access Rule Reference
- `DF_CD` — конфигурационные данные (опционально)
- `EF_UMPC` — максимальное энергопотребление UICC

### 9. Release 17+ Нововведения
- **Logical Secure Elements (LSE)** — множественные логические secure elements на одной карте
- **LSI (Logical Secure element Interface)** — логические соединения с точками доступа
- **Class D** (1.2V) поддержка
