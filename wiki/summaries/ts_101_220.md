# ETSI TS 101 220: ETSI Numbering System for Telecommunication Application Providers

> **Версия**: V15.1.0 (Release 15), май 2019
> **Комитет**: ETSI TC SCP (Smart Card Platform)
> **Страниц**: 41

## Обзор

**TS 101 220** — центральный реестр номеров ETSI/3GPP для UICC. Определяет: структуру AID (RID+PIX), выделение PIX для приложений (GSM, USIM, ISIM, Toolkit API), TAR-значения, COMPREHENSION-TLV теги (все 90+). Это **«телефонный справочник»** номеров UICC — не протокол, а реестр. ^[extracted]

## Ключевые разделы

### 1. Структура AID (§4)

```
<--------------------- AID (5-16 байт) --------------------->
┌──────────────────┬────────────────────────────────────────┐
│ RID (5 байт)     │ PIX (≤ 11 байт)                        │
└──────────────────┴────────────────────────────────────────┘

PIX структура:
  Digits 1-4:   Application code (напр. '0001' = GSM)
  Digits 5-8:   Country code (ITU-T E.164, right-justified, padded 'F')
  Digits 9-14:  Application provider code (E.118: Industry '89' + Card Issuer)
  Digits 15-22: Application provider field (8 hex digits, опционально)
```

### 6. TLV Data Objects (§7)

Специфицирует три формы TLV:
- **BER-TLV** (по ISO/IEC 8825-1)
- **COMPACT-TLV** (по ISO/IEC 7816-4)
- **COMPREHENSION-TLV** (ETSI-специфичный, с CR-флагом)

#### COMPREHENSION-TLV tag coding

| Первый байт | Формат |
|---|---|
| `01`-`7E` | Single byte (CR в бите 8) |
| `7F` | Three-byte format |
| `81`-`FE` | Single byte (CR в бите 8) |

CR = Comprehension Required (0 = понимание не обязательно, 1 = обязательно)

#### BER-TLV Tag Tables (выборочно)

**FCP Template (`62`)**: `80`=File Size, `82`=File Descriptor, `83`=File ID, `84`=DF Name, `8B`=Security Ref, `A5`=Proprietary

**Security Template (`AB`)**: `80`=AM Generic, `90`=SC ALWAYS, `97`=SC NEVER, `A4`=Control Ref Template, `A0`=OR, `AF`=AND

**Application Template (`61`)**: `4F`=AID, `50`=App Label, `51`=Path, `73`=Discretionary Template

**CAT Templates (`D0`-`DF`)**: `D0`=Proactive, `D1`=SMS-PP, `D3`=Menu Selection, `D4`=Call Control, `D6`=Event Download, `D7`=Timer Expiration

**COMPREHENSION-TLV Tags (90+ тегов)**: `01`=Cmd Details, `02`=Device ID, `05`=Alpha ID, `0D`=Text String, `0F`=Item, `13`=Location, `14`=IMEI, `19`=Event List, `24`=Timer ID, `31`=URL, `32`=Bearer, `60`=MAC... Полный список на страницах 17-20 спецификации.

### 7. PIX для Java Card API пакетов (Annex C, I, J, K, L)

**SIM Toolkit API packages**: ETSI app code=`0003`, Digits 5-8=`FFFF`, Digits 9-14=Industry `89`+`FFFF`, Digit 15=`1` (Java Card)

**UICC Toolkit API packages**: ETSI app code=`0005`

**(U)SIM API for Java Card (3GPP)**: 3GPP app code=`1005`

**ISIM API for Java Card**: app code=`1006`

**Contact Manager API**: app code=`1007`
