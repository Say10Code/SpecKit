# ETSI TS 102 226: Remote APDU Structure for UICC

> **Версия**: V16.3.0 (Release 16), октябрь 2022
> **Комитет**: ETSI TC SET
> **Страниц**: ~50

## Обзор

ETSI TS 102 226 определяет формат **Command Packet** для удалённой передачи APDU-команд на UICC через OTA (SMS-PP, CAT_TP). Определяет структуру пакета, TAR (Toolkit Application Reference), SPI (Security Parameters Indicator) и UICC-специфичные install parameters (теги `CA`, `EA`, `TA`). ^[extracted]

## Ключевые разделы

### 1. Command Packet Structure (§8)
```
CPL + SPI + KIC + KID + Secured Data
```

### 2. TAR — Toolkit Application Reference (§8.2.1.2)
Трёхбайтовый идентификатор, маршрутизирующий команды:
- `BF FF FF` — широковещательно всем STK-апплетам
- `XX YY ZZ` — конкретному апплету (задан в `CA` install param)
- `00 00 00` — RAM (Remote Application Management) → ISD
- `00 00 01` — RFM (Remote File Management)

### 3. SPI — Security Parameters Indicator (§8.2.1.1)
Кодирует:
- Используется ли шифрование
- Используется ли MAC
- Используется ли counter
- Какой алгоритм (DES/3DES/AES)

### 4. UICC Toolkit Install Parameters (§8.2.1.3)
Теги в INSTALL [for install]:

| Тег | Назначение |
|---|---|
| `CA` | SIM Toolkit parameters (Menu, Timers, TAR, Access Domain) |
| `EA` | eCAT parameters |
| `TA` | Terminal Application parameters |

### 5. Контроль параметров STK через `CA` (§8.2.1.3.2.1)
```
CA <len>
  01 <AccessDomainLen> <AccessDomain>
  00 <Priority>
  01 <MaxTimers>
  00 <MaxMenuTextLen>
  XX <MaxMenuEntries>
  YY × [<MenuEntry>]
  01 <MaxChannels>
  00 <MSL_Len> [<MSL_Data>]
  ZZ <TAR_Len> [<TAR_Data>...]
```
