# 3GPP TS 51.017: SIM Conformance Test Specification (Legacy GSM)

> **Версия**: V4.2.0 (Release 4)
> **Комитет**: 3GPP CT-T3
> **Файл**: `ts_151017v040200p.pdf`

## Обзор

**TS 51.017** — историческая спецификация тестирования совместимости (conformance) для GSM SIM-карт. Это предшественник TS 31.121. Тестирует исключительно GSM-окружение: файлы DF_GSM, CHV (Card Holder Verification), команду RUN GSM ALGORITHM. ^[inferred]

> TS 51.017 относится к TS 31.121 примерно как GSM 11.11 относится к TS 102 221 — это legacy-документ для 2G-тестирования.

## Отличия от TS 31.121 (UICC Conformance)

| Свойство | TS 51.017 (SIM) | TS 31.121 (UICC) |
|---|---|---|
| **Поколение** | 2G GSM | 3G/4G/5G (платформа UICC) |
| **Тестируемый объект** | SIM-карта (моно-приложение) | UICC (мульти-приложение) |
| **Базовая спецификация** | GSM 11.11 / TS 51.011 | TS 102 221 |
| **Протокол** | Только T=0 | T=0, T=1 |
| **Логические каналы** | Нет | MANAGE CHANNEL |
| **Классы напряжения** | Только 5V (Class A) | A/B/C/D |
| **Терминология безопасности** | CHV1/CHV2 (Card Holder Verification) | PIN1/PIN2 |
| **Аутентификация** | RUN GSM ALGORITHM (INS=88) | AUTHENTICATE (INS=88, разные P1) |
| **Таблица сервисов** | EF_SST (SIM Service Table) | EF_UST (USIM Service Table) |
| **Файл-контекст** | DF_GSM (7F20) | MF / ADF / DF |
| **Выбор приложения** | Нет (единственное приложение — GSM) | По AID (множественные приложения) |

## Ключевые тестируемые области

### 1. CHV (Card Holder Verification)
- Терминология CHV (не PIN!) — исторический артефакт GSM
- CHV1 (PIN1), CHV2 (PIN2), PUK1, PUK2
- VERIFY CHV (`INS=20`), CHANGE CHV (`INS=24`), DISABLE CHV (`INS=26`), ENABLE CHV (`INS=28`), UNBLOCK CHV (`INS=2C`)
- Счётчик оставшихся попыток в SW2 (как в UICC — '63 Cx')

### 2. RUN GSM ALGORITHM
- Уникальная команда GSM (`INS=88`, P1/P2=00/00)
- RAND (16 байт) → SRES (4 байта) + Kc (8 байт)
- Алгоритмы: COMP128-1/2/3
- **Не эквивалентна** AUTHENTICATE из UICC — это чисто GSM-функция

### 3. Файловая система DF_GSM
- EF_IMSI (0x6F07), EF_Kc (0x6F20), EF_PLMNsel (0x6F30)
- EF_SST (0x6F38) — SIM Service Table (до 35 сервисов)
- EF_LOCI (0x6F7E), EF_FPLMN (0x6F7B)
- EF_SPN (0x6F46), EF_AD (0x6FAD)

### 4. Команды специфичные для SIM
- SLEEP (`INS=FA`) — спящий режим (не поддерживается в UICC)
- INVALIDATE (`INS=04`), REHABILITATE (`INS=44`)
- SEEK (`INS=A2`) — предшественник SEARCH RECORD
