# 3GPP TS 31.101: UICC-Terminal Interface for 3GPP Applications

> **Версия**: V17.1.0 (Release 17), июль 2022
> **Комитет**: 3GPP TSG-CT6 (ранее TSG-T WG3)
> **Публикуется**: ETSI как TS 131 101
> **Страниц**: 39

## Обзор

3GPP TS 31.101 — это **надстройка над ETSI TS 102 221**, адаптирующая общий UICC-терминал интерфейс для 3GPP-приложений (USIM, ISIM, SIM). Документ **не дублирует** TS 102 221, а указывает, какие опции из базового стандарта **обязательны, рекомендованы или исключены** для 3GPP.

> 💡 **Мнемоника**: TS 102 221 = "как устроена любая UICC", TS 31.101 = "как устроена UICC для 3G/4G/5G".

## Ключевые отличия от TS 102 221

### 1. Электрический интерфейс
- **Class A (5V) не требуется** для 3GPP ME (кроме GSM ME) ^[extracted]
- UICC должна поддерживать минимум **два последовательных класса напряжения**, например AB или BC
- **Частота**: (F,D) = (512,32) **обязательна** для всех 3GPP UICC и терминалов; (512,64) рекомендована для MMS и высокоскоростных функций

### 2. ATR и Historical Bytes
- **TC1 в ATR**: терминал может отвергнуть UICC, если TC1 != 0 или 255 (extra guard time не требуется)
- ATR должен содержать global interface bytes для согласования протокола

### 3. Файловая система
- Все EF на уровне MF из TS 102 221 **обязательны**: `EF_ICCID`, `EF_DIR`, `EF_PL`, `EF_ARR`, `EF_UMPC`
- `DF_CD` — опционально
- EF_DIR **только AID**, без path object (выбор приложения — строго по AID)
- Минимальная тактовая частота приложения: **≤ 3 MHz** (tag '82' в FCP, значение '1E')

### 4. Безопасность и PIN
- **Два PIN**: PIN (обычный) и PIN2 (только в ADF)
- PIN/PIN2 кодируются на **8 байт** (ITU-T T.50 [5]), минимум 4 цифры
- Если пользователь вводит <8 цифр, ME добивает `'FF'`
- **UNBLOCK PIN**: всегда ровно 8 цифр
- Терминал **должен** поддерживать Universal PIN (key reference '11') и multi-verification
- Access rule reference: tag `'8B'` → EF_ARR (file ID + record number, или file ID + SEID + record number)

### 5. GSM vs USIM сессии
- **Взаимоисключающие**: активация USIM-сессии исключает GSM-сессию, и наоборот
- Команды с CLA='A0' (GSM) при активной USIM-сессии → `'6E 00'` (class not supported)
- Одновременно может быть активна **только одна** USIM-сессия

### 6. Команды — исключения для 3GPP
- `GET CHALLENGE` — **не требуется** 3GPP
- Все остальные generic команды из TS 102 221 обязательны
- Транспортно-ориентированные команды (GET RESPONSE и др. для T=0) — применяются

### 7. CAT (Card Application Toolkit)
- **Проактивный поллинг** в idle mode: STATUS каждые N секунд (N согласовывается)
- Если extended DRX — максимум из согласованного интервала и eDRX цикла сети
- Во время звонка: поллинг = UICC presence detection

### 8. Интерфейсы
- **Основной**: ISO/IEC 7816-3 совместимый (TS 102 221)
- **Опционально**: Inter-Chip USB (TS 102 600) — APDU напрямую поверх USB
- Контакт C6 для contactless (SWP) — **не требуется** 3GPP

## Модель 3GPP-приложения

```
Терминал (ME — Mobile Equipment)
  │ APDU команда
  ▼
UICC Platform (TS 102 221)
  ├─ USIM (TS 31.102)
  ├─ USAT (TS 31.111)  — USIM Application Toolkit
  ├─ ISIM (TS 31.103)
  └─ SIM (TS 51.011)   — legacy GSM
```

- **Application session**: от инициализации до де-селекции или разрыва связи
- **Application procedure**: последовательность command/response пар
- Терминал — primary role, UICC — secondary role
