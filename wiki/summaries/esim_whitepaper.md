# eSIM Whitepaper — The What and How of Remote SIM Provisioning (GSMA, 2018)

> **Издатель**: GSMA
> **Дата**: Март 2018
> **Страниц**: 21

## Обзор

Официальный whitepaper GSMA, объясняющий технологию eSIM (Remote SIM Provisioning) на концептуальном уровне. Описывает два решения: **M2M** (Machine-to-Machine) и **Consumer**. Не техническая спецификация, а введение в архитектуру и мотивацию. ^[extracted]

## Ключевые концепции

### Традиционная SIM vs eSIM

```
Традиционная SIM:            eSIM (RSP):
┌────────────┐               ┌─────────────────────┐
│SIM-карта (физическая)│       │ eUICC (встроенный чип)│
│ с профилем оператора│       │ ┌───────┬──────────┐│
└────────────┘               │ │Профиль│Профиль    ││
      ↓                      │ │Op A   │Op B      ││
Физическая замена при        │ └───────┴──────────┘│
смене оператора              └─────────────────────┘
                                    ↓
                           Профили загружаются
                           удалённо (OTA)
```

### Два решения GSMA

| Характеристика | M2M | Consumer |
|---|---|---|
| **Рынок** | IoT, промышленность | Смартфоны, планшеты, носимые устройства |
| **Управление** | Серверное (push) | Клиентское (pull, end user) |
| **Взаимодействие** | Без участия пользователя | Через UI устройства (LPA) |
| **Стандарты** | SGP.02 v4.x | SGP.22 v3.x |
| **Крипто-аутентификация** | PSK (Pre-Shared Key) | PKI (сертификаты) |
| **Ключевые компоненты** | SM-DP + SM-SR + eUICC | SM-DP+ + SM-DS + LPA + eUICC |

### Компоненты M2M

| Компонент | Роль |
|---|---|
| **SM-DP** (Data Preparation) | Подготовка, хранение, защита профилей; загрузка на eUICC |
| **SM-SR** (Secure Routing) | Управление статусом профилей (enable/disable/delete); защита канала |
| **eUICC** | Secure element с одним или несколькими профилями |

### Компоненты Consumer

| Компонент | Роль |
|---|---|
| **SM-DP+** (Data Preparation +) | Аналог SM-DP + SM-SR для consumer; создание и доставка профилей |
| **SM-DS** (Discovery Server) | «Доска объявлений» — SM-DP+ оставляет уведомления, LPA забирает |
| **LPA** (Local Profile Assistant) | Функции в устройстве: загрузка профиля + локальное управление (UI) |
| **eUICC** | Защищённый чип с поддержкой множественных профилей |

### Процесс: Consumer Solution

```
1. Пользователь заключает контракт с оператором
2. Оператор выдаёт QR-код (Activation Code) → адрес SM-DP+
3. Устройство сканирует QR-код → LPA подключается к SM-DP+
4. SM-DP+ и eUICC взаимно аутентифицируются (PKI)
5. Профиль загружается и устанавливается на eUICC
6. Профиль активируется → устройство в сети оператора
```

### Безопасность и сертификация

- **eUICC**: Common Criteria EAL4+ (Protection Profile PP0084)
- **Производство**: GSMA SAS (Security Accreditation Scheme) — SAS-UP для персонализации
- **Функциональная совместимость**: Тесты GSMA + GlobalPlatform
- **Сертификаты**: GSMA Certificate Issuer (CI) — корневой центр сертификации

### Ключевые различия решений

| M2M | Consumer |
|---|---|
| Нет end-user взаимодействия | End-user управляет профилями через UI |
| SM-SR как gateway | SM-DP+ напрямую к устройству |
| Один SM-SR на группу eUICC | Любой SM-DP+ может работать с любым eUICC |
| Использует BIP (SMS/CAT_TP/TCP) | Только IP (Wi-Fi или сотовая связь) |
| SM-SR Swap для смены оператора | Пользователь сам переключает профили |

### Преимущества eSIM

- **Для пользователей**: не нужно менять SIM-карты при смене оператора
- **Для операторов**: уход от логистики SIM-карт, новые каналы продаж (QR-коды)
- **Для производителей**: меньше места в устройстве, герметичность, упрощение supply chain
- **Для IoT**: удалённое управление миллионами устройств
