# US Patent 2013/0045737 A1 — Method for Instant Registration of a Roaming UE onto a Preferred VPLMN

> **Изобретатель**: Robert E. Manalo, North York (CA)
> **Патентообладатель**: TELUS Communications Company, Edmonton (CA)
> **Подана**: 17 августа 2011 | **Опубликована**: 21 февраля 2013
> **US Class**: 455/433 | **Int. Cl.**: H04W 8/06, H04W 64/00

## Обзор

Патент описывает метод **мгновенной и бесшовной перерегистрации** roaming-устройства (UE) с непредпочтительной гостевой сети (non-preferred VPLMN) на предпочтительную (preferred VPLMN) путём **обновления EF_LOCI** (байты MCCMNC) и отправки команды **REFRESH 3G Session Reset** в ME. ^[extracted]

Ключевая инновация: метод НЕ перезаписывает TMSI и Location Update Status — меняется ТОЛЬКО MCCMNC (байты 5-7 EF_LOCI, FID `0x6F7E`). Это заставляет ME считать новую сеть «последней зарегистрированной» и при REFRESH 3G Session Reset немедленно пытаться зарегистрироваться на ней.

---

## Как это работает

### Механизм (4 шага)

```
1. UE зарегистрирован на non-preferred VPLMN (напр. 302-880)
2. OTA/SMS обновляет EF_LOCI:
   - Байты 5-7 (MCCMNC): новый PLMN ID предпочтительной сети (напр. 302-221)
   - Байты 1-4 (TMSI): НЕ трогать (сохранить предыдущий TMSI!)
   - Байт 11 (Location Update Status): НЕ трогать (оставить "00" = updated)
3. UICC отправляет REFRESH 3G Session Reset → ME
4. ME выполняет MM Restart → читает EF_LOCI → видит "последний" PLMN 302-221 → регистрируется на нём
```

### 3GPP обоснование

Патент опирается на три требования 3GPP:

| Документ | Пункт | Суть |
|---|---|---|
| **TS 23.122** | §4.4.3.1 | При включении ME выбирает последний зарегистрированный PLMN (из EF_LOCI) |
| **TS 31.111** | §6.4.7 | REFRESH 3G Session Reset → сброс 3G-сессии + USIM Initialization + MM Restart |
| **TS 23.122** | §4.8 | MM Restart: ME ведёт себя как если бы SIM была вынута и вставлена заново |

### Ключевые ограничения (почему другие методы НЕ работают)

| Проблема | Почему предыдущие методы не работают |
|---|---|
| **Location Update Status ≠ 00** | ME не считает LOCI валидным → не пытается регистрироваться |
| **TMSI = FF FF FF FF** | Некоторые ME отказываются от регистрации без валидного TMSI |
| **REFRESH File Change Notification** | Не вызывает MM Restart на всех ME |
| **REFRESH UICC Reset** | На ME с Cold Reset вызывает power-cycle — пользователь думает что телефон сломался |
| **Только REFRESH 3G Session Reset** | Гарантирует MM Restart без power-cycle (≤ 30 секунд) |

## Результаты тестирования (COMPRION IT3)

Патент включает реальные логи UICC-ME обмена с COMPRION IT3 Proving Tool:

| Время (с) | Событие |
|---|---|
| 0.000 | ME включён, читает EF_LOCI (PLMN 302-880, TMSI валидный) |
| 31.892 | CS-регистрация на non-preferred 302-880 |
| 33.426 | PS-регистрация на 302-880 |
| **92.242** | **OTA-SMS: обновление EF_LOCI → MCCMNC = 302-221** |
| **97.453** | **UICC отправляет REFRESH 3G Session Reset** |
| 101.085 | ME перечитывает EF_LOCI (теперь 302-221) |
| 105.545 | CS-аутентификация на новой сети |
| 105.872 | PS-аутентификация на новой сети |
| **107.503** | **Регистрация на preferred 302-221 завершена** |

**Итого: 10.081 секунды от REFRESH до полной регистрации** (превышает 5-секундную цель TS 22.811, но значительно быстрее ручного переключения).

## Два способа реализации

### 1. OTA (коммерческое решение)
- SMS c обновлением EF_LOCI + REFRESH
- Используется оператором для управления роумингом

### 2. SIM Toolkit Application
- Пользователь выбирает страну из STK-меню
- Апплет обновляет EF_LOCI → отправляет REFRESH
- Пользователь видит меню, выбирает оператора → мгновенная перерегистрация

### Дополнительно: обновление OPLMNwAcT
Для предотвращения возврата на non-preferred сеть при HPPLMN rescan:
- Обновить EF_OPLMNwAcT (6F61) с preferred PLMN как высший приоритет
- Отправить REFRESH 3G Session Reset (содержащий EF_LOCI + EF_OPLMNwAcT)

## 14 патентных пунктов (Claims)

Основные пункты:
1. Метод перерегистрации UE через обновление MCCMNC в EF_LOCI + REFRESH 3G Session Reset
2. Регистрация в пределах 30 секунд
3. Без power-cycle (бесшовно)
4. ME соответствует TS 31.111 §6.4.7 + TS 23.122 §4.8 + TS 23.122 §4.4.3.1
5. Валидный TMSI в EF_LOCI
6. TMSI ≠ FF FF FF FF
7. Location Update Status = "updated"
8. Byte 11 FileID 6F7E = 00
9. Дополнительное обновление EF_OPLMNwAcT
10. Preferred PLMN как высший приоритет в OPLMNwAcT
11. REFRESH с EF_LOCI + EF_OPLMNwAcT в File List
12. Через коммерческое OTA-решение
13. Через SIM Toolkit Application
14. MCCMNC = байты 5-7 FileID 6F7E

## Применение в контексте ObsidianDB

Патент практически полезен для:
- **Steering of Roaming (SoR)**: самый простой способ заставить телефон переключиться на нужную сеть
- **STK-апплеты**: прямая реализация через SIM Toolkit (см. [[wiki/concepts/STK_Applet|STK Applet]])
- **Тестирование PLMN**: метод позволяет тестировать поведение телефона при смене PLMN без физического перемещения
- **Понимание EF_LOCI**: патент детально объясняет структуру и поведение LOCI (см. [[wiki/syntheses/sim_files_location|LOCI-семейство]])
- **REFRESH-команды**: практическая разница между REFRESH типами (File Change Notification vs 3G Session Reset vs UICC Reset)
