# ETSI TS 102 225: Secured Packet Structure for UICC

> **Версия**: V17.0.0 (Release 17), октябрь 2022
> **Комитет**: ETSI TC SET
> **Страниц**: 24

## Обзор

**TS 102 225** определяет формат **Secured Packet** — защищённого контейнера для передачи APDU-команд на UICC через ненадёжные каналы (SMS, CAT_TP, BIP). Это уровень безопасности **над** TS 102 226 (Command Packet). ^[inferred]

> 💡 **Архитектурно**: TS 102 226 = формат Command Packet (TAR, SPI). TS 102 225 = формат Secured Packet (шифрование + MAC поверх Command Packet).

## Ключевые концепции

### Secured Packet
```
┌──────────────┬──────────────┬──────────────┐
│ Security     │ Secured      │ MAC /        │
│ Header       │ Data         │ Signature    │
│ (SPI, KID...)│ (Command     │ (integrity)  │
│              │  Packet)     │              │
└──────────────┴──────────────┴──────────────┘
```

### Уровни безопасности (SPI)
Определяет какие механизмы применяются: шифрование (DES/3DES/AES), MAC, counter, signing.

### Применение
- **SMS-PP**: SMS-DELIVER содержит Secured Packet
- **CAT_TP**: BIP-канал с Secured Packet поверх
- **SCWS**: HTTPS + Secured Packet
