# 3GPP TS 31.102: USIM Application Characteristics

> **Версия**: V17.10.0 (Release 17), июль 2023
> **Комитет**: 3GPP TSG-CT6
> **Публикуется**: ETSI как TS 131 102
> **Страниц**: 368

## Обзор

3GPP TS 31.102 определяет **характеристики USIM-приложения** (Universal Subscriber Identity Module) — основного приложения на UICC для доступа к сетям 3GPP (UMTS, LTE, 5G). Это самая объёмная спецификация UICC — она перечисляет **все элементарные файлы (EF)** внутри ADF.USIM, их структуру, кодирование и access conditions. ^[extracted]

> 💡 **Мнемоника**: TS 102 221 = платформа UICC, TS 31.101 = 3GPP-адаптация платформы, **TS 31.102 = данные USIM** (все EF и процедуры).

## ADF.USIM — идентификация

- **AID**: `A0 00 00 00 87 10 02 FF xx xx xx` (xx — PIX, выделяемый 3GPP)
- Выбирается командой SELECT по AID
- После выбора ADF.USIM все команды с CLA=`00` адресованы USIM

## DF внутри ADF.USIM

### DF_5GS (5G Security)
- **5G-специфичные ключи и контексты безопасности**
- EF_5GAUTHKEYS, EF_5GS3GPPNSC, EF_5GSN3GPPNSC
- EF_SUCI_Calc_Info — расчёт SUCI (скрытого SUPI)
- EF_URSP — UE Route Selection Policy (5G slicing)
- EF_SOR-CMCI — Steering of Roaming — Connected Mode

### DF_PHONEBOOK
Телефонная книга с поддержкой e-mail, URI, second name, группировки:
- EF_PBR (Phone Book Reference), EF_ADN, EF_EMAIL, EF_PURI
- Синхронизация: EF_UID, EF_PSC, EF_CC

### DF_WLAN (I-WLAN)
Wi-Fi offloading: селекторы PLMN для WLAN, идентификаторы ре-аутентификации

### DF_ProSe (Proximity Services)
D2D (Device-to-Device): параметры direct discovery, direct communication, relay

### Другие DF
- **DF_GSM-ACCESS**: Kc, KcGPRS (legacy GSM-безопасность)
- **DF_HNB**: Home NodeB / CSG списки
- **DF_TV**: TV конфигурация
- **DF_SNPN**: Non-Public Networks
- **DF_V2X**: Vehicle-to-Everything (C-V2X)

## Управление сервисами: EF_UST

**USIM Service Table** (EF_UST) — битовая маска, определяющая, какие сервисы USIM поддерживает. Терминал читает EF_UST при инициализации и активирует/деактивирует соответствующие функции.

Примеры бит сервисов (100+ в Release 17):
- Service 1: Local Phone Book
- Service 2: Fixed Dialling Numbers (FDN)
- Service 12: Service Provider Name
- Service 17: Advice of Charge
- Service 33: eMLPP
- Service 58: MMS
- Service 86: ProSe
- Service 99: 5GS
- Service 128+: Beyond Release 17

## Процедуры USIM

### Инициализация (Clause 5.1)
1. SELECT по AID → активация USIM-сессии
2. Чтение EF_UST → определение поддерживаемых сервисов
3. Чтение EF_EST → определение включённых сервисов
4. Чтение EF_AD → Administrative Data (MNC/MCC, длина PIN, ...)
5. Чтение EF_SPN, EF_PNN, EF_OPL → отображение имени оператора

### Безопасность (Clause 5.2)
- **AUTHENTICATE**: 3G AKA, EAP-AKA', 5G AKA
- **GSM Security**: Kc (A3/A8), KcGPRS (GPRS)
- **GBA**: Generic Bootstrapping Architecture
- **Key derivation**: CK, IK, KASME, K_AUSF, K_SEAF

### Подписка (Clause 5.3)
- Телефонная книга (ADN, FDN, SDN)
- SMS хранение
- Advice of Charge (ACM, ACMmax)
- PLMN селекция (user/operator preferred)
- Запрещённые PLMN (FPLMN)

## Структура ADF.USIM

```
ADF.USIM (AID: A0 00 00 00 87 10 02 ...)
├── EF_IMSI        ← Главный идентификатор
├── EF_Keys        ← CK + IK (Circumstances для 3G/4G)
├── EF_UST         ← USIM Service Table
├── EF_SPN         ← Service Provider Name
├── EF_AD          ← Administrative Data
├── EF_FPLMN       ← Запрещённые сети
├── ... ещё ~100 EF ...
│
├── DF_5GS         ← 5G Security
│   ├── EF_5GAUTHKEYS
│   ├── EF_5GS3GPPNSC
│   └── EF_URSP
├── DF_PHONEBOOK   ← Контакты
│   ├── EF_PBR
│   └── EF_ADN
├── DF_GSM-ACCESS  ← Legacy GSM
│   ├── EF_Kc
│   └── EF_KcGPRS
├── DF_WLAN        ← Wi-Fi offload
├── DF_ProSe       ← D2D
├── DF_TV          ← ТВ
└── DF_V2X         ← Автомобили
```
