# 3GPP TS 51.011: SIM-ME Interface (Legacy GSM)

> **Версия**: V4.15.0 (Release 4)
> **Комитет**: 3GPP CT-T3
> **Файл**: `ts_151011v041500p.pdf`

## Обзор

**TS 51.011** — преемник GSM 11.11 в рамках 3GPP. Определяет физические и логические характеристики интерфейса SIM-ME для Java Card-совместимых SIM-карт. Это мост между классической SIM (GSM 11.11, только 5V, только T=0) и полноценной UICC (TS 102 221, мульти-приложение, T=1). ^[inferred]

> Историческая цепочка: GSM 11.11 (1995) → TS 51.011 (1999+) → TS 102 221 (UICC). TS 51.011 перенёс спецификацию SIM в 3GPP и добавил поддержку Java Card.

## Ключевое содержание

### 1. Физические характеристики
- **ID-1 SIM** (85.6 × 54 × 0.76 мм) и **Plug-in SIM** (25 × 15 × 0.76 мм)
- Контакты: C1=Vcc, C2=RST, C3=CLK, C5=GND, C6=Vpp (программирование), C7=I/O
- Напряжение: Class A (5V ± 10%) — только 5V для legacy GSM
- CLK: 1-5 MHz (типично 3.57 или 4.92 MHz)

### 2. ATR и протокол
- ATR до 33 байт, direct/inverse convention
- PTS (Pin Type Selection) — исторический предшественник PPS
- Протокол передачи: **только T=0** (полудуплекс, байт-ориентированный)
- Скорость по умолчанию: 9600 bps; возможна переконфигурация через ATR
- Procedure bytes: '60', '61', '6C', '9F' — стандартный T=0 механизм

### 3. Файловая система
- MF (Master File, 0x3F00) — корень
- DF_GSM (0x7F20) — GSM Application (основной контейнер)
- DF_TELECOM (0x7F10) — телефонная книга и SMS
- Типы EF: transparent, linear fixed, cyclic (как в UICC)
- Доступ: SELECT по FID (2 байта), путь не поддерживается

### 4. Карта команд для Java Card
- Стандартные команды: SELECT, STATUS, READ/UPDATE BINARY, READ/UPDATE RECORD, SEEK, INCREASE, GET RESPONSE
- Команды безопасности: VERIFY CHV, CHANGE CHV, DISABLE CHV, ENABLE CHV, UNBLOCK CHV
- GSM-специфичные: RUN GSM ALGORITHM (INS=88), SLEEP (INS=FA)
- Команды жизненного цикла: INVALIDATE (INS=04), REHABILITATE (INS=44)
- **Java Card**: SIM выполнена как Java Card апплет с GSM-специфичными расширениями

### 5. Безопасность
- CHV (Card Holder Verification) вместо PIN-терминологии UICC
- CHV1 = PIN1, CHV2 = PIN2 (счётчик попыток, блокировка, PUK)
- Access conditions: ALWays, CHV1, CHV2, ADM, NEVer
- RUN GSM ALGORITHM: RAND (16 байт) → SRES (4 байта) + Kc (8 байт)

### 6. SIM Service Table (EF_SST, 0x6F38)
Битовая маска сервисов (35+ сервисов в Phase 2+):
- Service 1: CHV1 disable
- Service 2: ADN (Abbreviated Dialling Numbers)
- Service 4: SMS storage
- Service 5: Advice of Charge
- Service 7: PLMN Selector
- Service 29: CBMIR (Cell Broadcast Message Identifier Range)
- ... и другие

## Отличия от TS 102 221

| Свойство | TS 51.011 (SIM) | TS 102 221 (UICC) |
|---|---|---|
| **Платформа** | SIM (моно-приложение) | UICC (мульти-приложение) |
| **Приложения** | 1 (GSM) | Множественные (USIM, ISIM, ...) |
| **Протокол** | T=0 | T=0, T=1, USB |
| **Логические каналы** | Нет | 0-19 |
| **Выбор приложения** | Фиксировано (только GSM) | По AID |
| **Напряжение** | 5V | A/B/C/D |
| **Файловая система** | MF-DF-EF | MF-DF-ADF-EF |
| **EF_DIR** | Нет | Да (0x2F00) |
| **Таблица сервисов** | EF_SST | EF_UST |
| **Аутентификация** | RUN GSM ALGORITHM (только COMP128) | AUTHENTICATE (UMTS/EPS/5G AKA) |
| **Безопасность** | CHV | PIN |
| **Java Card** | SIM как апплет | UICC как платформа с апплетами |
