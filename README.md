# ObsidianDB

**Самонаполняемая техническая вики по телеком-спецификациям с AI-оркестрацией.**

Ты говоришь: «скачай TS 31.102 и напиши по ней статью» — ObsidianDB находит спецификацию на 3GPP FTP, скачивает, извлекает эталонный текст, создаёт структурированное саммари, связывает с остальной базой знаний и проверяет факты по первоисточнику. Всё за ~3 минуты.

> 📂 **Где что лежит**: [[STRUCTURE.md]] — полная карта проекта (Core / Data / Build / Generated).

---

## Что умеет

| Возможность | Как это работает |
|---|---|
| 🔍 **Найти и скачать** | `python -m _pipeline download 31.102` — 3GPP FTP + WhatTheSpec API. Без авторизации, без ручного поиска |
| 📄 **Обработать спецификацию** | 3-tier extraction: .docx → TXT+таблицы (0.2 сек) / PDF → структурированный Markdown+JSON (GPU) / PDF → плоский текст (fallback) |
| 📝 **Написать саммари** | Author v2 читает документ один раз и создаёт summary + 3-8 concepts + entities за один проход. Все страницы знают друг о друге |
| 📚 **Написать статью** | Researcher собирает источники из всей базы знаний (summaries, concepts, спецификации) + WebSearch → synthesis или research-статья на 15-50 KB |
| 🔬 **Провести ресерч** | `/research 5G NR MAC scheduling` — Researcher + WebSearch → глубокий разбор с Mermaid-диаграммами и сравнительными таблицами |
| ✅ **Проверить факты** | Reviewer v3 сверяет каждое утверждение с эталонным текстом спецификации (гибридный Pass 1: TXT Grep + JSON lookup + MD read) |
| 🔗 **Поддерживать связность** | Linker анализирует граф знаний, находит сирот, добавляет перекрёстные ссылки. 0 битых ссылок в любой момент времени |
| 📊 **Следить за качеством** | `/lint --deep` строит полный граф wikilinks и проверяет 8 категорий метрик. Quality Score 98/100 (A) |

## Архитектура

| Слой | Что внутри | Размер |
|---|---|---|
| **AI Intelligence** | 8 агентов + 7 скиллов + 6 includes | 43 граф-узла |
| **Processing Engine** | `_pipeline/` (speckit: download + 3-tier extract) + 13 скриптов | 165 узлов |
| **Engineering Docs** | Архитектура, планы, отчёты, диаграммы | 228 узлов |
| **Data** | wiki/ (130 стр.), Specifications/ (74 PDF), specs-extracted/ (237 файлов) | 237+ |
| **Cache** | `.speckit/` — метаданные 3GPP/ETSI спецификаций (SQLite) | — |

8 агентов работают как команда:
- **SpecDownloader** → находит и скачивает
- **Librarian** → сортирует и каталогизирует
- **Author v2** → пишет страницы знаний (Batch: 1 вызов вместо N)
- **SpecExtractor v3** → извлекает эталонный текст (3-tier: .docx/Docling/PyPDF2)
- **Linker** → поддерживает граф связей
- **Reviewer v3** → проверяет факты по первоисточнику
- **Researcher** → глубокие исследования (15-50 KB)
- **Formatter** → экспорт в HTML
