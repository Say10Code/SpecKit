# Skill: lint-wiki
# Trigger: /lint

Проверка здоровья вики-хранилища. Используется после каждого изменения в wiki/.

## Синтаксис

```
/lint              # Базовая проверка: битые ссылки, сироты, frontmatter
/lint --deep       # Полный граф-анализ связности (аудит)
```

## Workflow — базовый режим

1. **Битые ссылки**: найди все `[[wiki/...]]` которые никуда не ведут
2. **Сиротские страницы**: страницы без входящих wikilinks (исключая index.md)
3. **Качество frontmatter**: проверь все поля — tags, type, status, created, updated, sources
4. **Противоречия**: конфликтующие утверждения между страницами (разные FID для одного EF)
5. **Пробелы**: темы без покрытия

## Workflow — режим `--deep` (полный граф-анализ)

Выполни базовую проверку, затем **запусти скрипт**:

```bash
python "D:\ObsidianDB\_tech\scripts\audit_connectivity.py"
```

Скрипт анализирует ВЕСЬ граф wikilinks и создаёт отчёт: `_tech/reports/connectivity-audit-YYYY-MM-DD.md`

### Что проверяет `--deep` (дополнительно к базовому)

| Проверка | Порог | Что значит |
|---|---|---|
| **Битые ссылки** | 0 | wikilinks на несуществующие страницы |
| **Сироты** | 0 | 0 входящих wikilinks |
| **Слабые страницы** | <3 in / <3 out | Недостаточно интегрированы в граф |
| **Изолированные кластеры** | 0 | Группы страниц без внешних связей |
| **Мостовые страницы** | — | Страницы, соединяющие разделы wiki/ |
| **Cross-ref пробелы** | — | Страницы на одну тему без взаимных ссылок |
| **Плотность связей** | avg >5 | Среднее число in/out ссылок на страницу |
| **Матрица типов** | — | Связность по типам страниц (concept→summary, ...) |

### Интерпретация результатов

```
0 битых + 0 сирот + <5 слабых + 0 кластеров = HEALTHY
1-5 проблем = NEEDS ATTENTION
>5 проблем = DEGRADED — требуется Linker
```

## Инструменты

- Glob: список всех `.md` в `wiki/`
- Grep: поиск `[[wiki/...]]` паттернов
- Read: проверка frontmatter
- `audit_connectivity.py`: граф-анализ (networkx + BFS/DFS)

## Отчёт

После базовой проверки выведи сводку: общее количество страниц, битых ссылок, сирот, страниц без тегов, проблем frontmatter.

После `--deep` — сошлись на `_tech/reports/connectivity-audit-YYYY-MM-DD.md` и подсвети ключевые находки: топ-3 проблемы связности с конкретными страницами.

## Периодичность

- **Базовый `/lint`** — после КАЖДОГО изменения в wiki/
- **`/lint --deep`** — еженедельно (или по запросу)
- **Авто-напоминание**: PostToolUse hook срабатывает на Edit/Write в wiki/
