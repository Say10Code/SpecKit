# Spec Registry — полный реестр спецификаций

> **Назначение**: ЕДИНСТВЕННЫЙ source of truth для маппинга «тема → номер спецификации».
> **Используется**: Researcher, SpecDownloader, /research skill, /spec-download skill.
> **Правило**: перед любым research'ем — прочитай этот файл. Если тема не найдена → вероятно, её нет в 3GPP (GSMA, ISO, другое).
> **Обновление**: `python -m _pipeline registry update` — дополняет описания из WhatTheSpec API.

---

## Легенда

| Приоритет | Когда читать |
|---|---|
| **P0** | Ядро домена — скачаны и извлечены, всегда в работе |
| **P1** | Часто нужны для кросс-доменных исследований |
| **P2** | Связанные темы — нужны реже, но важны для контекста |
| **P3** | Специфичные / редко используемые / справочные |

## Как читать реестр

```
Запрос: «UE capability release detection через UECapabilityInformation»

1. Ключевые слова → домены:
   "UE capability" → UE Capability секция
   "UECapabilityInformation" → NR RRC (38.331), LTE RRC (36.331)
   "release detection" → NAS (24.501/24.301) + capability (38.306/36.306)

2. По реестру:
   → Секция «UE Capability»:
     38.306 — NR UE radio access capabilities (FeatureSet, bandCombinationList)        [P0]
     36.306 — LTE UE radio access capabilities (featureGroupIndicators)                [P0]
   → Секция «NR / 5G RAN»:
     38.331 — NR RRC (UECapabilityEnquiry/Information, request filter)                 [P1]
   → Секция «LTE / E-UTRAN»:
     36.331 — LTE RRC (UECapabilityTransfer, inter-RAT)                                [P1]
   → Секция «NAS»:
     24.501 — 5G NAS (5GMM capability, UE security capabilities)                       [P2]
     24.301 — EPS NAS (UE network capability IE, MS network capability)                [P2]
   → Секция «5G Core»:
     23.501 — 5GS Architecture (N1 mode, interworking, release support)                [P3]
   → Итого: 7 спецификаций

3. Проверить наличие:
   specs-extracted/ → .venv\Scripts\python.exe -m _pipeline download <номер> если нет
```

## Как добавлять спецификации

1. Определить серию и домен по номеру: 38.xxx → NR/RAN, 24.xxx → NAS, etc.
2. Проверить WhatTheSpec API: `.venv\Scripts\python.exe -m _pipeline metadata fetch <номер>`
3. Добавить строку в соответствующую секцию с правильным приоритетом:
   - P0: ядро домена, используется постоянно
   - P1: часто нужна для кросс-доменных исследований
   - P2: связанная тема, важна для контекста
   - P3: специфичная / редко используемая
4. Если домен новый → создать секцию с перекрёстными ссылками
5. Запустить `.venv\Scripts\python.exe -m _pipeline registry update` для синхронизации названий
