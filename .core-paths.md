## Исключения — Core-файлы внутри Data-директорий

Эти файлы физически лежат внутри Data, но являются Core (версионируются, не генерируются автоматически):

| Что | Где | Почему в Data | Как попадает в git |
|---|---|---|---|
| **.category-map.md** | `Specifications/.category-map.md` | Нужен Librarian'у для сортировки `Specifications/`. Должен лежать рядом с файлами которые сортирует | `!Specifications/.category-map.md` в .gitignore |
| **INDEX.md** | `specs-extracted/INDEX.md` | Реестр форматов извлечённых текстов. Должен лежать рядом с индексируемыми файлами | `!specs-extracted/INDEX.md` в .gitignore |
| **.gitkeep** | `Specifications/!INCOMING/.gitkeep` | Маркер пустой директории для git | `!Specifications/!INCOMING/.gitkeep` |
| **.gitkeep** | `Specifications/!double/.gitkeep` | Маркер пустой директории для git | `!Specifications/!double/.gitkeep` |

## Как определить что ты в Core

Три признака:

1. **Имя начинается с `_` или `.`** → Core (`.claude/`, `_pipeline/`, `.obsidian/`)
2. **Файл в корне с описанием проекта** → Core (`CLAUDE.md`, `README.md`, `Roadmap.md`, `STRUCTURE.md`, `.core-paths.md`)
3. **Исключение из `.gitignore`** → Core-файл внутри Data (`.category-map.md`, `INDEX.md`)

Всё остальное — не Core.
