# EF.SPN и EF.PNN: Спецификация элементарных файлов в UICC

## Введение

EF.SPN (Elementary File Service Provider Name) и EF.PNN (Preferred Network Name) — это элементарные файлы, используемые в SIM-картах для хранения информации о сетях. Они определены в ETSI TS 102 221 и играют ключевую роль в управлении подключением к мобильным сетям.

## EF.SPN (Service Provider Name)

### Описание
EF.SPN содержит имя провайдера услуг, который предоставляет доступ к сетям. Это поле используется для идентификации оператора, к которому привязана SIM-карта.

### Структура данных
- **Файл идентификатор**: `0x6F07` (2 байта)
- **Длина данных**: 2 байта (макс. 64 символа)
- **Данные**: UCS2 кодирование (16 бит на символ, 128 байт)
- **Общее размер файла**: 132 байта (включая заголовок)

### Примеры кодирования
- "Vodafone" → UCS2 коды для каждого символа (16 бит/символ)
- "Мобильные Сети" → UCS2 коды для кириллических символов (16 бит/символ)

## EF.PNN (Preferred Network Name)

### Описание
EF.PNN хранит предпочтительное имя сети, которое отображается пользователю при подключении к мобильной сети. Это поле позволяет персонализировать отображение сетей на устройстве.

### Структура данных
- **Файл идентификатор**: `0x6F08` (2 байта)
- **Длина данных**: 2 байта (макс. 64 символа)
- **Данные**: UCS2 кодирование (16 бит на символ, 128 байт)
- **Общее размер файла**: 132 байта (включая заголовок)

### Примеры кодирования
- "Мои Сети" → UCS2 коды для кириллических символов (16 бит/символ)
- "My Networks" → UCS2 коды для латиницы (16 бит/символ)

## Annex A (нормативное): UCS2 кодирование Alpha-полей для файлов на UICC

### Общие положения
Согласно ETSI TS 102 221, все Alpha-поля (альфанумерические данные) в файлах UICC должны кодироваться в UCS2. Это обеспечивает поддержку международных символов и локализации.

### Требования к кодировке
- **Кодировка**: UCS2 (16-битные коды Unicode)
- **Поддержка**: Все символы из Unicode, включая кириллицу, арабский, китайский и другие
- **Порядок байтов**: Little Endian (младший байт первый)
- **Битовая распределение**: 16 бит на символ (2 байта)

### Примеры кодирования
1. **Латиница**: "Hello World" → UCS2 коды для каждого символа (16 бит/символ)
2. **Кириллица**: "Привет, мир!" → UCS2 коды для кириллических символов (16 бит/символ)
3. **Арабский**: "مرحبا بالعالم" → UCS2 коды для арабских символов (16 бит/символ)
